---
title: "Quick start"
description: "Run your Node.js app with nodemon and learn the essential commands in a few minutes."
---

<Steps>

<Step title="Install nodemon">

Install nodemon globally so it's available from any directory:

```bash
npm install -g nodemon
```

See the [installation page](/installation) for yarn, pnpm, and local install options.

</Step>

<Step title="Run your app">

Replace `node` with `nodemon` when starting your application:

```bash
nodemon server.js
```

nodemon starts your process and watches for file changes. When you save a file, it automatically restarts.

You can pass arguments to your script exactly as you would with `node`:

```bash
nodemon ./server.js localhost 8080
```

Lines prefixed with `[nodemon]` are nodemon's own status messages. All other output comes from your application.

**Auto-detection from `package.json`**

If you run `nodemon` without specifying a script, it reads your `package.json` and uses the `main` field as the entry point. It also checks `scripts.start` (as of nodemon 1.1.x). This means in a project with a `main` field set, you can simply run:

```bash
nodemon
```

</Step>

<Step title="Pass node flags">

You can forward flags to the underlying `node` process. For example, to start the inspector for debugging:

```bash
nodemon --inspect ./server.js
```

Any argument nodemon does not recognize is passed through to the executing command.

</Step>

<Step title="Restart manually">

While nodemon is running, type `rs` and press Enter to trigger a manual restart without stopping nodemon:

```
rs
```

This is useful when you want to force a restart without changing a file — for example, after updating an environment variable.

</Step>

<Step title="Stop nodemon">

Press **Ctrl+C** to stop nodemon and exit.

</Step>

</Steps>

<Tip>
  Run `nodemon --help` to see a full list of available flags and options directly in your terminal.
</Tip>
