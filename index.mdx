Nodemon is a drop-in replacement for the `node` command that watches your project files and restarts your application automatically whenever you save a change. No code modifications needed — just swap `node` for `nodemon` and get instant feedback during development.

## How it works

<Steps>
  <Step title="Install nodemon">
    Install globally via npm, yarn, or pnpm — or add it as a dev dependency in your project.

    ```bash
    npm install -g nodemon
    ```
  </Step>
  <Step title="Replace node with nodemon">
    Use nodemon exactly like `node`. Pass your entry file and any arguments your app needs.

    ```bash
    nodemon server.js
    ```
  </Step>
  <Step title="Edit your code">
    Make changes to any watched file. Nodemon detects the change and automatically restarts your process.
  </Step>
  <Step title="Customize as needed">
    Add a `nodemon.json` config file to control which files are watched, which are ignored, restart delays, and more.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Zero configuration" icon="bolt">
    Works out of the box with any Node.js app. Just replace `node` with `nodemon`.
  </Card>
  <Card title="Flexible file watching" icon="eye">
    Watch specific directories, file extensions, or glob patterns using `--watch` and `--ext`.
  </Card>
  <Card title="Non-Node.js scripts" icon="file-code">
    Run Python, Ruby, or any other language with `--exec`. Nodemon monitors the right file types automatically.
  </Card>
  <Card title="Programmatic API" icon="plug">
    Integrate nodemon into your build tools, scripts, or custom tooling via its Node.js module API.
  </Card>
  <Card title="Event hooks" icon="bell">
    Trigger shell commands or custom scripts on `start`, `restart`, `crash`, and `quit` events.
  </Card>
  <Card title="Docker & network support" icon="server">
    Legacy polling mode (`-L`) enables reliable watching in containers and over mounted network drives.
  </Card>
</CardGroup>
